工单回调

1. 功能说明

  • 工单状态变化后，美团会主动将变更内容推动接入方。若回调失败，则会进行重试，重试间隔为1min,2min,4min,8min,16min,32min,64min,128min，若还是失败则消息丢弃

  • 为防止因异常情况导致的回调失败，造成双方系统的订单状态和信息不一致，建议接入方在回调基础上，查询工单进度接口，来确保信息的一致性。

2. 请求地址

接入方提供

3. 服务协议

HTTP POST

4. 参数

参数 是否必须 字段类型 参数描述
appKey 必须 String 令牌
appSecret 必须 String 数字签名
timestamp 必须 Long 时间戳
customerId 必须 Integer 商ID
orderSource 必须 Integer 详见概述：订单来源
eventCode 必须 Integer 详见概述：订单状态及事件类型
workCaseId 必须 Integer 美团工单号
operateLogId 必须 Integer 工单操作日志id

5. 返回值

{
    "code": 200,
    "success": true,
    "msg": "成功",
    "traceId": 5857893724238462783,
    "data": null
}

6.返回值说明

返回结果，美团会校验是否成功，如果校验失败会发起重试
